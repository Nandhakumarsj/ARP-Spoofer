What is ARP?

  • ARP stands for Address Resolution Protocol
  • Operates at Layer 2 (Data Link Layer) of OSI model
  • Translates IP addresses (Layer 3) to MAC addresses (Layer 2)
  • Essential for communication in Local Area Networks (LANs)
  • Defined in RFC 826 (1982)
  • Stateless protocol with no built-in authentication

ARP Working Principle

  1. Step 1: Host A needs to communicate with Host B
  2. Step 2: Host A broadcasts ARP Request: "Who has IP X.X.X.X?"
  3. Step 3: All devices receive the broadcast
  4. Step 4: Host B recognizes its IP and sends ARP Reply
  5. Step 5: Host B replies with its MAC address (unicast)
  6. Step 6: Host A caches the IP-to-MAC mapping in ARP table

What is a Network Packet?

Definition

  • A packet is a small unit of data transmitted over a network
  • Contains structured information organized in layers
  • Each layer adds its own header with control information
  • Also called PDU (Protocol Data Unit) at different layers

Three Main Components

1. Header
  • Contains control information (source, destination, type)
  • Added by each network layer
  • Allows routers and switches to understand packet
2. Payload (Data)
  • The actual user data/content
  • Messages, files, streams, etc.
  • Size varies based on protocol
3. Trailer/Footer (Optional)
  • Error checking information (CRC/Checksum)
  • Ensures data integrity
  • May contain framing information

ARP Packet Structure

An ARP packet is carried within an Ethernet frame at the Data Link Layer.
It contains only the necessary information to map IP addresses to MAC addresses.
Total size: 28 bytes (for IPv4 over Ethernet)

Section 1: Hardware & Protocol Info (8 bytes)

  • Hardware Type (HTYPE) - 16 bits
    Specifies network type (1 = Ethernet)
  • Protocol Type (PTYPE) - 16 bits
    Specifies protocol (0x0800 = IPv4)
  • Hardware Address Length (HLEN) - 8 bits
    Length of MAC address (6 bytes for Ethernet)
  • Protocol Address Length (PLEN) - 8 bits
    Length of IP address (4 bytes for IPv4)

Section 2: Operation (2 bytes)

  • Operation Code - 16 bits
    1 = ARP Request
    2 = ARP Reply
    Tells recipients what action to take

Section 3: Sender Information (10 bytes)

  • Sender Hardware Address (SHA) - 48 bits (6 bytes)
    MAC address of the packet sender
  • Sender Protocol Address (SPA) - 32 bits (4 bytes)
    IPv4 address of the packet sender

Section 4: Target Information (10 bytes)

  • Target Hardware Address (THA) - 48 bits (6 bytes)
    MAC address of the target host
    (unknown in requests, filled in replies)
  • Target Protocol Address (TPA) - 32 bits (4 bytes)
    IPv4 address of the target host

Packet Layers and Encapsulation

Complete network transmission showing encapsulation from application to physical layer

How Data Travels Through Network Layers

Application Layer: User data ("GET /index.html")
↓ Add Transport Header
Transport Layer: TCP Segment (adds port numbers, sequence)
↓ Add Network Header
Network Layer: IP Packet (adds source/destination IP)
↓ Add Data Link Header
Data Link Layer: Ethernet Frame (adds MAC addresses)
↓ Convert to signals
Physical Layer: Bits on the wire (0s and 1s)

⚠️ Note: ARP packets skip the Transport and Application layers,
working directly at the Data Link Layer!

Why is ARP Vulnerable?

• No Authentication Mechanism

ARP accepts responses without verifying sender identity

• Stateless Protocol

Devices cache any ARP reply, even unsolicited ones

• Trust-Based Design

Originally designed for efficiency, not security

• No Encryption

All ARP messages sent in plaintext

• Automatic Cache Updates

ARP entries overwritten without verification

Classification of ARP Attacks

1. ARP Spoofing/Poisoning

• Sending falsified ARP messages to LAN

• Associates attacker's MAC with legitimate IP

2. Man-in-the-Middle (MITM)

• Intercepting traffic between two hosts

• Eavesdropping and data manipulation

3. Denial of Service (DoS)

• Flooding network with ARP packets

• Linking multiple IPs to single MAC

4. Session Hijacking

• Stealing session IDs via ARP spoofing

• Unauthorized access to systems

How ARP Spoofing Works

Phase 1: Scanning

Attacker scans network for IP/MAC addresses

Phase 2: Spoofing

Sends forged ARP replies to target devices

Phase 3: Interception

Redirects traffic through attacker's machine

Phase 4: Exploitation

Steals data, modifies traffic, or blocks communication

How ARP Spoofing Works Presented by Nandhakumar S J | Slide 10 of 18

MITM Attack via ARP Poisoning

  • Attacker positions between victim and gateway
  • Both victim and gateway poisoned simultaneously
  • Traffic flows through attacker transparently
  • Enables credential theft, data interception
  • Often remains undetected by users

Detecting ARP Attacks

Manual Detection

  • Monitor ARP cache for duplicate MAC addresses
  • Check for unsolicited ARP replies
  • Analyze ARP traffic patterns
  • Use command: arp -a (Windows) or arp -n (Linux)

Automated Tools

  • Wireshark - Packet analysis
  • XArp - Real-time ARP monitoring
  • Arpwatch - ARP cache monitoring
  • Snort - Intrusion Detection System
  • OpUtils - Network monitoring

Network Features

  • Dynamic ARP Inspection (DAI)
  • Port Security
  • DHCP Snooping

Prevention & Mitigation Techniques

Network Level

  • Enable Dynamic ARP Inspection (DAI) on switches
  • Implement Static ARP entries for critical devices
  • Use VLANs for network segmentation
  • Configure port security

Protocol Level

  • Deploy encrypted protocols (HTTPS, SSH, IPSec)
  • Use VPNs for sensitive communications
  • Implement 802.1X authentication

System Level

  • Regular security updates and patches
  • Deploy endpoint protection
  • Enable packet filtering
  • Use anti-spoofing software
